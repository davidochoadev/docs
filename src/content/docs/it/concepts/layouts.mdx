**Layouts** sono [componenti Astro](/it/core-concepts/astro-components/) utilizzati per fornire una struttura UI riutilizzabile, come un modello di pagina.

Un tipico componente di layout Astro fornisce [pagine Astro, Markdown o MDX](/it/core-concepts/astro-pages/) con:

- un **contenitore di pagina** (`<html>`, `<head>` e `<body>`)
- un [**`<slot />`**](/it/core-concepts/astro-components/#slots) per specificare dove dovrebbe essere iniettato il contenuto individuale della pagina.

Ma, non c'√® nulla di speciale in un componente di layout! Possono [accettare props](/it/core-concepts/astro-components/#component-props) e [importare ed utilizzare altri componenti](/it/core-concepts/astro-components/#component-structure) come qualsiasi altro componente Astro. Non √® nemmeno necessario che forniscano un contenitore di pagina completo e possono invece essere utilizzati come modelli parziali dell'UI.

I componenti di layout sono comunemente collocati in una directory `src/layouts` nel tuo progetto per l'organizzazione, ma questo non √® un requisito; puoi scegliere di collocarli ovunque nel tuo progetto. ### Markdown Layout Props

Un layout Markdown/MDX avr√† accesso alle seguenti informazioni tramite `Astro.props`:

- **`file`** - Il percorso assoluto di questo file (ad esempio `/home/user/projects/.../file.md`).
- **`url`** - Se √® una pagina, l'URL della pagina (ad esempio `/it/guide/contenuto-markdown`).
- **`frontmatter`** - tutti i frontmatter dal documento Markdown o MDX.
  - **`frontmatter.file`** - Lo stesso della propriet√† di livello superiore `file`.
  - **`frontmatter.url`** - Lo stesso della propriet√† di livello superiore `url`.
- **`headings`** - Una lista di intestazioni (`h1 -> h6`) nel documento Markdown o MDX con metadati associati. Questa lista segue il tipo: `{ depth: number; slug: string; text: string }[]`.
- **(Solo Markdown) `rawContent()`** - Una funzione che restituisce il documento Markdown grezzo come stringa.
- **(Solo Markdown) `compiledContent()`** - Una funzione che restituisce il documento Markdown compilato in una stringa HTML.

Un esempio di post di blog Markdown potrebbe passare il seguente oggetto `Astro.props` al suo layout:
